### 4.1. AI Spatial Highlighting (Bounding Boxes)

**The Challenge:** While Vision LLMs (like GPT-4o) are great at answering questions, they are historically imprecise at returning exact pixel coordinates on massive 8K resolution images.
**The Solution (OCR-Semantic Hybrid):** We use the VLM to identify the exact *text or label* of the object, and then cross-reference that string against the precise bounding boxes generated by Azure Document Intelligence (from Phase 1).

```python
def find_bounding_box(sheet_id: str, target_text: str, ocr_database: dict):
    """
    Searches the OCR layout data (saved in Phase 1) for the target_text 
    and returns normalized coordinates (0.0 to 1.0).
    """
    for word in ocr_database[sheet_id]["words"]:
        # Simple string match (in production, use fuzzy matching/Levenshtein)
        if target_text.lower() in word["content"].lower():
            polygon = word["polygon"] #[x1, y1, x2, y2, x3, y3, x4, y4]
            
            # Convert polygon to bounding box
            x_coords = [p[0] for p in polygon]
            y_coords = [p[1] for p in polygon]
            
            return BoundingBox(
                text=word["content"],
                x=min(x_coords),
                y=min(y_coords),
                width=max(x_coords) - min(x_coords),
                height=max(y_coords) - min(y_coords)
            )
    return None
```

const drawHighlights = (highlights: BoundingBox[], viewer: OpenSeadragon.Viewer) => {

        // OpenSeadragon uses a normalized Viewport coordinate system where Width is always 1.0
        // and Height is ImageHeight / ImageWidth.
        const imageSize = viewer.world.getItemAt(0).getContentSize();
        const normX = box.x / imageSize.x;
        const normY = box.y / imageSize.x; // Divided by X to maintain aspect ratio
        const normWidth = box.width / imageSize.x;
        const normHeight = box.height / imageSize.x;

        const rect = new OpenSeadragon.Rect(normX, normY, normWidth, normHeight);

        viewer.addOverlay({
            element: overlayElement,
            location: rect,
            placement: OpenSeadragon.Placement.CENTER
        });

        // Smoothly zoom the camera to the highlight
        viewer.viewport.fitBoundsWithConstraints(rect);
    });
};

### 4.2. Automated Revision Comparison (Pixel Diffing)

In construction, a "Delta" or "Bulletin" is issued when a sheet changes. Identifying what changed in a massive drawing is notoriously difficult for humans. We will use OpenCV to generate a "Diff Map".

Here is the Python script to generate a color-coded difference image (`revision_diff.py`):

```python
import cv2
import numpy as np
from skimage.metrics import structural_similarity as ssim

def generate_blueprint_diff(path_v1: str, path_v2: str, output_path: str):
    print(f"Comparing {path_v1} and {path_v2}...")

    # 1. Load images in grayscale
    img1 = cv2.imread(path_v1, cv2.IMREAD_GRAYSCALE)
    img2 = cv2.imread(path_v2, cv2.IMREAD_GRAYSCALE)

    # Note: If PDFs were scanned or shifted, you would add an alignment step here 
    # using ORB/SIFT feature matching. Assuming digitally generated PDFs, they are aligned.
    
    # Ensure images are the exact same size
    if img1.shape != img2.shape:
        img2 = cv2.resize(img2, (img1.shape[1], img1.shape[0]))

    # 2. Compute Structural Similarity Index (SSIM)
    # score gives the percentage match, diff is the image array of differences
    score, diff = ssim(img1, img2, full=True)
    diff = (diff * 255).astype("uint8")

    # 3. Threshold the difference to get exact pixels that changed
    _, thresh = cv2.threshold(diff, 200, 255, cv2.THRESH_BINARY_INV | cv2.THRESH_OTSU)

    # 4. Find contours of the changes
    contours, _ = cv2.findContours(thresh, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

    # 5. Create a visual overlay map
    # We will make v2 the base image (converted to BGR so we can add color)
    diff_map = cv2.cvtColor(img2, cv2.COLOR_GRAY2BGR)

    # 6. Color Coding:
    # To find what was ADDED vs REMOVED, we check the pixel values in the original images
    # within the changed regions.
    
    for c in contours:
        # Filter out tiny noise (e.g., anti-aliasing artifacts)
        if cv2.contourArea(c) < 10:
            continue
            
        x, y, w, h = cv2.boundingRect(c)
        
        # Region of Interest in both images
        roi1 = img1[y:y+h, x:x+w]
        roi2 = img2[y:y+h, x:x+w]
        
        # In blueprints, white is background (255), black is ink (0)
        # If ink is present in v2 but not v1 -> ADDITION (Color it Blue)
        # If ink is present in v1 but not v2 -> DELETION (Color it Red)
        
        # Draw bounding boxes around changes
        cv2.rectangle(diff_map, (x, y), (x + w, y + h), (0, 0, 255), 2) # Red outline for visibility

    cv2.imwrite(output_path, diff_map)
    print(f"Diff map saved to {output_path} with similarity score: {score:.2f}")
    
    # Phase 1 integration: Send this output_path to pyvips to generate a DZI for the viewer!
    return output_path
```

### 4.3. Smart Callout Hyperlinking

Blueprints contain callouts like `"See Section 4 / A-302"`. We want these to act like hyperlinks on the web.

#### The Regex + Graph Logic
During **Phase 1 (Ingestion)**, after Azure OCR runs, we execute a post-processing sweep over the text blocks.

```python
import re

def create_sheet_hyperlinks(ocr_data, all_sheet_numbers):
    """
    Scans OCR data for standard architectural callout patterns and links them to target sheets.
    all_sheet_numbers is a list of all known sheets in the database (e.g.,['A-101', 'A-302', ...])
    """
    links =[]
    
    # Common pattern for detail callouts: "4 / A-302", "DETAIL 4/A-302", "A-302"
    callout_pattern = re.compile(r'\b([A-Z]{1,2}[-\s]*\d{1,3})\b')

    for word in ocr_data["words"]:
        text = word["content"]
        matches = callout_pattern.findall(text)
        
        for match in matches:
            clean_match = match.replace(" ", "") # Normalize "A - 302" to "A-302"
            
            # If the referenced sheet actually exists in our archive project
            if clean_match in all_sheet_numbers:
                links.append({
                    "target_sheet": clean_match,
                    "bounding_box": word["polygon"]
                })
                
    return links # Save this array to PostgreSQL
```

**Implementing the Rectangle**

I'm now thinking about how to render the hyperlinks as rectangles using the data structure. It looks like I need to normalize the coordinates to work with OpenSeadragon's coordinate system. I have to transform the bounding box coordinates, which are originally based on the OCR output, into normalized values for OpenSeadragon's view. This involves calculating min/max X and Y coordinates to get width and height and then converting them into normalized values relative to the image size.


```tsx
const renderHyperlinks = (links: CalloutLink[], viewer: OpenSeadragon.Viewer, onSheetSelect: (sheetNum: string) => void) => {
    // OpenSeadragon uses a normalized coordinate system where image width is 1.0
    const imageSize = viewer.world.getItemAt(0).getContentSize();

    links.forEach((link, index) => {
        // 1. Convert Polygon to Bounding Box
        // Azure OCR returns polygons as[x1, y1, x2, y2, x3, y3, x4, y4] or array of {x,y}
        // Assuming array of [x, y] for simplicity here:
        const xCoords = link.bounding_box.map(p => p[0]);
        const yCoords = link.bounding_box.map(p => p[1]);
        
        const minX = Math.min(...xCoords);
        const minY = Math.min(...yCoords);
        const width = Math.max(...xCoords) - minX;
        const height = Math.max(...yCoords) - minY;

        // 2. Normalize coordinates
        const normX = minX / imageSize.x;
        const normY = minY / imageSize.x; // Note: OSD height is scaled relative to width
        const normWidth = width / imageSize.x;
        const normHeight = height / imageSize.x;

        const rect = new OpenSeadragon.Rect(normX, normY, normWidth, normHeight);

        // 3. Create the interactive HTML overlay
        const linkElement = document.createElement("div");
        linkElement.id = `hyperlink-${index}`;
        linkElement.className = "absolute cursor-pointer border-2 border-transparent hover:border-blue-500 hover:bg-blue-500 hover:bg-opacity-30 transition-all rounded";
        linkElement.title = `Jump to Sheet ${link.target_sheet}`;

        // 4. Handle Click (Teleport to new sheet)
        // Note: In OpenSeadragon, native DOM clicks can conflict with panning/dragging.
        // We use a MouseTracker for clean interaction.
        new OpenSeadragon.MouseTracker({
            element: linkElement,
            clickHandler: (event) => {
                event.preventDefaultAction = true; // Prevent OSD from zooming
                console.log(`Navigating to ${link.target_sheet}...`);
                onSheetSelect(link.target_sheet); // Trigger React state update to load new DZI
            }
        });

        // 5. Add to Viewer
        viewer.addOverlay({
            element: linkElement,
            location: rect
        });
    });
};
```
